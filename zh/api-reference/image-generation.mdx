`/v1/images/generations` 根据文本提示词生成图片，`/v1/images/edits` 根据输入图片和提示词编辑图片。两个端点均兼容 OpenAI Images API，并支持 `gpt-image-2` 等兼容模型。

<Warning>
  Gemini 图片模型使用 Gemini 原生 `generateContent` 端点，不能通过本端点调用。请参阅 [Gemini 图片模型操作手册](/zh/guides/image-generation#使用-gemini-图片模型)。
</Warning>

## 端点

```
POST https://router.flatkey.ai/v1/images/generations
```

## 请求

### 请求头

| 请求头 | 值 |
|---|---|
| `Authorization` | `Bearer $FLATKEY_API_KEY` |
| `Content-Type` | `application/json` |

### 请求体参数

<ParamField body="model" type="string" required>
  与 OpenAI Images API 兼容的图片模型 ID，例如 `"gpt-image-2"`。
</ParamField>

<ParamField body="prompt" type="string" required>
  对要生成图片的文本描述。提示词越详细，通常生成效果越好。
</ParamField>

<ParamField body="n" type="integer">
  要生成的图片数量。默认值：1。最大值因模型而异。
</ParamField>

<ParamField body="size" type="string">
  图片尺寸。常用值包括 `"256x256"`、`"512x512"`、`"1024x1024"`、`"1792x1024"`、`"1024x1792"`。支持情况因模型而异。
</ParamField>

<ParamField body="quality" type="string">
  图片质量设置。可选值：`"standard"` 或 `"hd"`。默认值：`"standard"`。仅部分模型支持。
</ParamField>

<ParamField body="response_format" type="string">
  `"url"`（默认）返回图片的临时 URL；`"b64_json"` 返回 Base64 编码的图片数据。
</ParamField>

## 请求示例

<CodeGroup>
```python python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["FLATKEY_API_KEY"],
    base_url="https://router.flatkey.ai/v1",
)

response = client.images.generate(
    model="gpt-image-2",
    prompt="A futuristic city skyline at dusk, neon lights reflecting on wet streets, cyberpunk style",
    size="1024x1024",
    n=1,
)

print(response.data[0].url)
```
```javascript node
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.FLATKEY_API_KEY,
  baseURL: "https://router.flatkey.ai/v1",
});

const response = await client.images.generate({
  model: "gpt-image-2",
  prompt: "A futuristic city skyline at dusk, neon lights reflecting on wet streets, cyberpunk style",
  size: "1024x1024",
  n: 1,
});

console.log(response.data[0].url);
```
```bash curl
curl https://router.flatkey.ai/v1/images/generations \
  -H "Authorization: Bearer $FLATKEY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "A serene forest clearing with morning mist and sunbeams",
    "size": "1024x1024",
    "n": 1
  }'
```
</CodeGroup>

## 响应

```json
{
  "created": 1710000000,
  "data": [
    {
      "url": "https://cdn.openai.com/...",
      "revised_prompt": "A futuristic city skyline at dusk..."
    }
  ]
}
```

### 响应字段

<ResponseField name="created" type="integer">
  请求完成时的 Unix 时间戳。
</ResponseField>

<ResponseField name="data" type="array">
  生成的图片对象数组。
  <Expandable title="图片对象">
    <ResponseField name="url" type="string">生成图片的临时 URL。请及时下载，URL 会过期。</ResponseField>
    <ResponseField name="b64_json" type="string">Base64 编码的图片数据，仅当 `response_format` 为 `"b64_json"` 时提供。</ResponseField>
    <ResponseField name="revised_prompt" type="string">经过安全修订后实际发送给模型的提示词。</ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  图片 URL 是临时地址，会在短时间后过期。请下载并保存需要长期保留的图片。
</Warning>

## 编辑图片

`/v1/images/edits` 使用 `multipart/form-data` 上传待编辑图片和编辑指令。

### 端点

```
POST https://router.flatkey.ai/v1/images/edits
```

### 请求头

| 请求头 | 值 |
|---|---|
| `Authorization` | `Bearer $FLATKEY_API_KEY` |
| `Content-Type` | `multipart/form-data`；使用 curl 时由 `-F` 自动生成并附带 boundary |

### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `model` | string | 是 | 图片模型 ID，例如 `gpt-image-2`。 |
| `prompt` | string | 是 | 对图片的编辑指令。 |
| `image` | file | 是 | 待编辑图片，支持 PNG、JPEG 和 WebP。多图可使用 `image[]` 或 `image[0]`、`image[1]`。 |
| `mask` | file | 否 | 用于局部重绘的蒙版；透明区域是需要重绘的区域。 |
| `size` | string | 否 | 输出图片尺寸，支持值因模型而异。 |
| `quality` | string | 否 | 输出画质档位，支持值因模型而异。 |

### 请求示例

```bash
curl https://router.flatkey.ai/v1/images/edits \
  -H "Authorization: Bearer $FLATKEY_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=在图片中央添加一颗黄色的小星星" \
  -F "image=@input.png" \
  -F "mask=@mask.png"
```

`mask` 是可选参数。省略它即可根据提示词编辑整张图片。不要手动添加 `Content-Type` 请求头；curl 会根据 `-F` 参数自动设置正确的 multipart boundary。

### 响应示例

```json
{
  "created": 1710000000,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAA...",
      "revised_prompt": "Edit the provided image: add a small yellow star in the center...",
      "url": ""
    }
  ]
}
```

编辑后的图片位于 `data[0].b64_json`，内容为 Base64 编码的图片数据。输入图片也会计入模型用量，因此图片编辑通常比纯文本生成图片消耗更多用量。
