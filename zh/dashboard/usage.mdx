Flatkey **使用日志**让你完整查看账号发出的每次 API 请求。每条日志都会显示使用的模型、输入和输出 Token 数量、从余额中扣除的费用以及时间戳。利用这些信息可以分析支出模式、排查错误并核对费用。

## 打开使用日志

<Steps>
  <Step title="打开控制台">
    登录 [console.flatkey.ai](https://console.flatkey.ai/usage-logs/common?lng=zh)，然后点击左侧边栏中的 **使用日志**。
  </Step>
  <Step title="设置日期范围">
    使用顶部的日期选择器，按日、周、月或自定义范围筛选日志。默认显示最近 7 天。
  </Step>
  <Step title="按模型或密钥筛选">
    使用 **模型名称** 和 **API 密钥** 筛选器查看特定模型或密钥。这适合比较模型费用或审计某个集成。
  </Step>
</Steps>

## 每条日志显示什么

| 字段 | 说明 |
|---|---|
| **时间戳** | Flatkey 收到请求的时间 |
| **模型** | 使用的模型 ID，例如 `gpt-4o-mini`、`claude-sonnet-4-5` |
| **输入 Token** | 提示词中的 Token 数量 |
| **输出 Token** | 模型回复中的 Token 数量 |
| **费用** | 本次请求从余额中扣除的金额 |
| **API 密钥** | 使用的密钥，以名称和脱敏前缀显示 |
| **状态** | 请求成功还是失败 |

## 理解费用

每次请求显示实际收取的费用，其中包含：
- 模型的输入和输出价格（按 Flatkey 折扣价）
- 自动应用的当前充值赠送额度

所有请求费用之和等于该时间段内消耗的总余额。

## 监控余额

当前余额显示在使用日志顶部和 **钱包** 页面。余额低于你设置的阈值时，会收到邮件提醒。可在 **设置 → 通知** 中配置提醒。

<Note>
  请求完成时会实时扣除余额。如果请求在提供商侧失败，具体扣费取决于失败前已经处理的 Token 数量，可能仍会产生部分费用。
</Note>

## 导出日志

导出用量数据：
1. 设置需要的日期范围和筛选条件。
2. 点击右上角的 **导出**。
3. 选择 CSV 或 JSON 格式。
4. 文件会下载到浏览器。

导出的日志包含控制台中显示的全部字段，可以导入电子表格或财务工具。

## 高用量提醒

为防止支出失控：
- 在 **设置 → 计费** 中设置 **月度支出上限**，达到上限后会阻止后续请求。
- 使用 **分组** 为每个密钥或应用设置额度。

<Tip>
  开发期间使用单独密钥，并设置较小的支出额度（例如 $5），可以防止意外的高费用请求耗尽生产余额。
</Tip>
