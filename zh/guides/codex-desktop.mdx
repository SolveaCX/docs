Codex 桌面端与 Codex CLI 读取同一份用户级 Codex 配置。你可以将 Flatkey 添加为自定义模型供应商，而无需替换官方 ChatGPT 或 Codex 登录。

## 前置条件

- 已安装 Codex 桌面端
- 已在桌面应用中完成至少一次官方 ChatGPT 或 Codex 登录
- Flatkey API 密钥——[在 Flatkey 控制台中创建](https://console.flatkey.ai/keys?lng=zh)
- [Flatkey 模型目录](/zh/reference/model-list)中的受支持模型 ID
- 如果使用 CC Switch 配置，需要 CC Switch v3.17.0 或更高版本，建议使用最新版

## 手工配置

### 1. 设置 API 密钥

将密钥保存在用户环境变量中，不要写入配置文件。

<CodeGroup>
```powershell Windows PowerShell
[Environment]::SetEnvironmentVariable(
  "FLATKEY_API_KEY",
  "sk-fk-...",
  "User"
)
```
```bash macOS
launchctl setenv FLATKEY_API_KEY "sk-fk-..."
```
```bash Linux
export FLATKEY_API_KEY="sk-fk-..."
```
</CodeGroup>

更改环境变量后，请完全退出 Codex 桌面端，再重新打开。

### 2. 更新 `config.toml`

打开用户级 Codex 配置文件：

| 平台 | 配置文件路径 |
|---|---|
| Windows | `%USERPROFILE%\.codex\config.toml` |
| macOS 和 Linux | `~/.codex/config.toml` |

更新顶层的 `model` 和 `model_provider` 设置。如果其中任一键已存在，请替换它的值，不要追加重复键。保留所有无关设置。然后添加下方供应商配置表；如果 `[model_providers.flatkey]` 已存在，请更新该表，不要重复创建：

```toml
model = "gpt-5.4"
model_provider = "flatkey"

[model_providers.flatkey]
name = "Flatkey"
base_url = "https://router.flatkey.ai/v1"
env_key = "FLATKEY_API_KEY"
wire_api = "responses"
```

本文撰写时，`gpt-5.4` 已在 Flatkey 模型目录中提供。切换到其他模型 ID 前，请先查阅模型目录。

<Warning>
  官方凭据可能保存在 `auth.json` 或操作系统凭据存储中。配置 Flatkey 时，切勿编辑、替换、覆盖、导出或分享任一凭据存储。不要向其中任何一处粘贴 Flatkey API 密钥。模型和供应商路由由 `config.toml` 控制。
</Warning>

### 3. 重启并验证

完全退出 Codex 桌面端，重新打开并发送一条简单提示。打开 [Flatkey 用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)，确认模型、token 数量、延迟和费用。

## 使用 CC Switch 配置

[CC Switch](https://ccswitch.io) 是第三方配置管理器，不是 Flatkey 产品。你也可以从 [CC Switch GitHub 仓库](https://github.com/farion1231/cc-switch)下载。本教程已在 CC Switch v3.17.0 中验证；较新版本的界面文字可能不同。

手工配置和 CC Switch 是两种可替代的配置方式。请只用其中一种方式控制当前供应商。Codex 桌面端和 Codex CLI 共用 macOS 与 Linux 上的 `~/.codex/config.toml`，或 Windows 上的 `%USERPROFILE%\.codex\config.toml`。CC Switch 会控制这份共享文件中的模型和供应商设置。请保留无关设置和供应商。

### 1. 完成一次官方登录

切换供应商前，请打开 Codex 桌面端并完成一次官方 ChatGPT 或 Codex 登录。桌面应用需要这次有效的官方会话来访问账户和模型目录。

官方凭据保存在 `auth.json` 或操作系统凭据存储中。两者都属于敏感信息。

### 2. 保留官方登录以便直接切换

在 CC Switch 中打开**设置** > **通用** > **Codex App 增强**，然后启用**非接管切换时保留官方登录**。这是 v3.17.0 中的准确标签。启用后，直接切换供应商时不会丢弃 Codex 桌面端仍需使用的官方登录。

<Warning>
  切勿为了复制内容而打开 `auth.json`，也不要替换、覆盖、分享该文件，或向其中粘贴 Flatkey API 密钥。不要在截图或 CC Switch 供应商导出文件中暴露 Flatkey API 密钥。如果任一凭据已经暴露，请立即轮换或撤销。
</Warning>

### 3. 将 Flatkey 添加为自定义供应商

打开 CC Switch 的 **Codex** 面板并添加**自定义供应商**。请使用下表中的值。本教程不依赖内置的 Flatkey 供应商条目。

| 供应商字段 | 值 |
|---|---|
| 供应商名称 | `Flatkey` |
| API 密钥 | `sk-fk-...` |
| API 端点 | `https://router.flatkey.ai/v1` |
| 默认模型 | `gpt-5.4`——请在 [Flatkey 模型目录](/zh/reference/model-list)中确认当前准确的模型 ID |
| 上游格式 | **Responses（原生）** |

### 4. 配置模型

将供应商的**默认模型**设置为你要使用的准确 Flatkey 模型 ID。本文撰写时可以使用 `gpt-5.4` 作为示例。使用其他 ID 前，请先查看 [Flatkey 模型目录](/zh/reference/model-list)。

如果你的 CC Switch 版本提供**获取模型**功能，请用它加载当前模型 ID。要让模型出现在 Codex `/model` 菜单中，请将其添加到供应商的模型映射。你也可以将准确 ID 设置为供应商的**默认模型**，或在 Codex CLI 中通过 `codex --model <id>` 传入该 ID。验证 Codex 桌面端时，即使选择器没有列出自定义模型，也应将当前供应商和默认模型视为预期路由。发送请求后，请通过 Flatkey 用量日志确认实际路由。

Flatkey 原生支持 Responses API。请关闭**本地路由**；此配置不需要它。

### 5. 保存、启用并重启

保存自定义供应商，并为 Codex 启用它。不要从共享的 `config.toml` 中删除无关设置或供应商。

完全退出 Codex 桌面端。确认没有旧的 Codex 桌面端进程残留，再重新打开应用，让它读取新的供应商设置。

### 6. 验证实际路由

新建一个上下文较小的对话，并发送一条最简提示。然后打开 [Flatkey 用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)，确认该请求的模型、token 数量、延迟和费用。

保留官方登录并不代表请求会由 OpenAI 计费。共享 `config.toml` 中的当前供应商和模型设置决定实际路由。Flatkey 用量日志是请求已到达 Flatkey 的权威验证依据。

系统提示、工具、对话历史、附件和命令输出都可能增加输入 token，因此输入量可能高于可见提示文本。更高的输入 token 数量可能增加计费费用。

### 如果模型不可见

Codex 桌面端可能显示官方模型目录，而不在选择器中显示自定义模型。界面中缺少自定义模型并不能证明路由失败。

1. 确认官方 ChatGPT 或 Codex 登录仍然有效。
2. 确认已启用**设置** > **通用** > **Codex App 增强** > **非接管切换时保留官方登录**。
3. 确认 Flatkey 供应商已启用，而且其**默认模型**使用准确的模型 ID。
4. 检查供应商的模型映射。如果当前 CC Switch 版本提供**获取模型**，并且你希望模型出现在 `/model` 菜单中，请使用该功能。
5. 确认共享 `config.toml` 中是预期的当前供应商和模型，不要更改无关设置。
6. 完全退出所有 Codex 桌面端进程，再重新打开应用。
7. 发送一条最简提示，并在 [Flatkey 用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)中确认实际路由。

### 切回官方供应商

在 CC Switch 中启用官方 Codex 供应商。如果你使用手工配置，请在 `config.toml` 中恢复之前的顶层 `model` 和 `model_provider` 值。保留无关设置和供应商表。不要改动 `auth.json` 或操作系统凭据存储中的官方凭据。完全退出 Codex 桌面端，确认没有旧进程残留，再重新打开。无需删除 Flatkey 自定义供应商。

### CC Switch 故障排查

| 问题 | 检查项 |
|---|---|
| 官方登录缺失或已过期 | 通过 Codex 桌面端重新登录，然后在启用 Flatkey 前打开**非接管切换时保留官方登录** |
| 身份验证失败或 Flatkey 密钥被拒绝 | 确认 API 密钥仍有效，并已填入自定义供应商的 API 密钥字段；如果密钥已经暴露，请轮换或撤销 |
| 端点或 Responses 报错 | 使用 `https://router.flatkey.ai/v1`，选择 **Responses（原生）**，并关闭**本地路由** |
| 出现 `Model not found`、自定义模型不可见或缺少 `/model` 映射 | 从[模型目录](/zh/reference/model-list)复制准确 ID，将其设为**默认模型**，并在可用时使用**获取模型**或模型映射 |
| 用量日志显示错误的供应商或模型 | 启用 Flatkey 自定义供应商，并确认其默认模型以及共享 `config.toml` 中的当前值 |
| 更改未生效 | 完全退出 Codex 桌面端，结束所有残留的 Codex 桌面端进程，然后重新打开 |
| Flatkey 余额不足 | 充值或改用有可用额度的 Flatkey 密钥，然后重试一条最简提示 |
| 误开本地路由 | 关闭**本地路由**，因为 Flatkey 可以接收原生 Responses 请求 |
| 用量日志中没有请求 | 新建上下文较小的对话并发送一条最简提示，然后重新检查当前供应商、API 端点和[用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh) |
| 切换后官方登录消失 | 通过 Codex 桌面端重新登录并启用登录保留设置；切勿替换或覆盖 `auth.json` |
