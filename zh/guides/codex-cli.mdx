OpenAI Codex CLI 是一款在终端中运行的 AI 编程助手。设置两个环境变量，即可将 Codex CLI 的全部请求路由到 Flatkey，并按 Flatkey 的优惠价格计费，GPT 模型用量通常可节省 40% 以上。

## 前置条件

- 已安装 Codex CLI：`npm install -g @openai/codex`
- 拥有 Flatkey 账号和 API 密钥——[在此获取](https://console.flatkey.ai/sign-up?lng=zh)

## 配置

运行 Codex CLI 前，设置以下环境变量：

```bash
export OPENAI_BASE_URL="https://router.flatkey.ai/v1"
export OPENAI_API_KEY="sk-fk-..."
```

也可以将它们添加到 shell 配置文件（`~/.bashrc`、`~/.zshrc`），使设置永久生效：

```bash ~/.zshrc
export OPENAI_BASE_URL="https://router.flatkey.ai/v1"
export OPENAI_API_KEY="sk-fk-..."
```

## 使用 CC Switch 配置

[CC Switch](https://ccswitch.io) 是面向编程助手的第三方桌面配置管理器，不属于 Flatkey，也没有内置 Flatkey 预设。以下步骤以当前 CC Switch 界面为准。

上方的手动 `OPENAI_BASE_URL` 和 `OPENAI_API_KEY` 配置方法与 CC Switch 是两种替代方案。测试 CC Switch 时，只需清除指向其他供应商或密钥的过期 `OPENAI_BASE_URL` 或 `OPENAI_API_KEY` 值。shell 继承的这些值可能覆盖 CC Switch 中启用的供应商，或与其发生冲突。

### 1. 安装 CC Switch

从 [ccswitch.io](https://ccswitch.io) 或其[官方 GitHub 仓库](https://github.com/farion1231/cc-switch)下载 CC Switch。安装适用于你操作系统的版本，然后打开应用。

### 2. 打开 Codex 供应商面板

在 CC Switch 中选择 **Codex**。此操作会打开供应商面板，用于管理新 Codex 会话使用的配置。

![在 CC Switch 中打开 Codex 供应商面板并选择添加供应商按钮](/images/guides/cc-switch/zh/codex-provider-list.png)

### 3. 将 Flatkey 添加为自定义供应商

选择**自定义供应商（Custom Provider）**，并使用以下值添加供应商：

| 字段 | 值 |
|---|---|
| 供应商名称 | `flatkey` |
| API 密钥 | 你的 Flatkey 密钥，例如 `sk-fk-...` |
| API 请求地址 | `https://router.flatkey.ai/v1` |
| 默认模型 | 截图中使用 `gpt-5.6-sol`。使用前请在 Flatkey [模型目录](/zh/reference/model-list)中确认并复制当前准确 ID。 |
| 上游格式 | `Responses（原生）` |

![在 CC Switch 中填写当前 Flatkey 供应商字段](/images/guides/cc-switch/zh/codex-provider-form.png)

<Warning>
  请将 API 密钥视为敏感信息。不要在截图或导出、共享的供应商配置中包含该密钥。如果密钥泄露，请在 Flatkey 中撤销或轮换该密钥，然后在 CC Switch 中替换。
</Warning>

Flatkey 原生支持 Responses API。此配置不需要 CC Switch **本地路由**，请将其保持关闭。

### 4. 配置模型

使用 Flatkey [模型目录](/zh/reference/model-list)中的准确模型 ID 作为默认模型。请勿使用显示名称或猜测模型 ID。

模型映射只影响 Codex `/model` 菜单中显示的条目。点击**获取模型列表**，然后映射你想显示的准确模型 ID。即使准确模型 ID 未被映射或未显示，你仍可将其设为供应商的**默认模型**，或通过 `codex --model <id>` 直接传入。

![选择原生 Responses 上游格式并配置 Codex 模型映射](/images/guides/cc-switch/zh/codex-model-mapping.png)

### 5. 保存并启用 Flatkey

保存供应商，然后在 Codex 供应商面板中启用 **Flatkey**。启动 Codex 前，请确认当前启用的供应商是 Flatkey。

![确认 Flatkey Codex 供应商处于使用中](/images/guides/cc-switch/zh/codex-provider-active.png)

### 6. 重启并验证

退出现有 Codex 会话并结束其进程。关闭运行 Codex 的特定终端窗口或标签页，不要关闭其他无关终端。打开一个不会继承过期相关 `OPENAI_BASE_URL` 或 `OPENAI_API_KEY` 值的新终端。进入一个较小或空目录并启动新的 Codex 会话。如果配置了模型映射，可以先打开 `/model` 确认条目已经出现；这只能验证映射是否加载，不能证明请求已经通过 Flatkey。随后发送一个最小请求。

![在新 Codex CLI 会话中确认 Flatkey 模型出现在模型列表中](/images/guides/cc-switch/zh/codex-cli-verify.png)

随后打开 Flatkey [用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)，确认实际请求。检查模型、输入和输出 token、延迟及费用。

<Note>
  智能体请求可能包含系统提示词、工具定义、对话历史、文件和命令结果。连接测试时请保持较小的上下文，以便轻松核对 token 数量和费用。
</Note>

### 切换回其他供应商

在 CC Switch 中打开 Codex 供应商面板，选择并启用你要使用的供应商。结束现有 Codex 会话和进程，然后打开一个不会继承过期相关 `OPENAI_BASE_URL` 或 `OPENAI_API_KEY` 值的新终端。启动新会话。

### CC Switch 故障排查

| 问题 | 解决方法 |
|---|---|
| 身份验证失败或密钥无效 | 重新输入 Flatkey 密钥，并确认它以 `sk-fk-` 开头。再次测试前，只清除过期的 `OPENAI_API_KEY` 值。 |
| API 请求地址或连接错误 | 将**API 请求地址**准确设置为 `https://router.flatkey.ai/v1`。检查新 shell 中是否存在过期的 `OPENAI_BASE_URL`。 |
| 找不到模型或 `/model` 中没有模型 | 从[模型目录](/zh/reference/model-list)复制准确 ID。使用**获取模型列表**和模型映射。映射只影响 `/model`；请将准确 ID 设为供应商的**默认模型**，或通过 `codex --model <id>` 传入。 |
| 请求使用了错误的供应商 | 返回 Codex 供应商面板并启用 **Flatkey**。检查手动设置的 `OPENAI_*` 值是否指向其他供应商。 |
| 更改未影响当前会话 | 退出现有 Codex 会话并结束其进程。关闭运行 Codex 的特定终端窗口或标签页，不要关闭其他无关终端。打开一个不会继承过期相关 `OPENAI_BASE_URL` 或 `OPENAI_API_KEY` 值的新终端，然后启动新会话。 |
| 余额不足 | 充值 Flatkey 余额，然后重试最小请求。 |
| **本地路由**被误启用 | 关闭**本地路由**。Flatkey 原生接受 Responses 格式。 |
| 用量日志中没有请求 | 发送新的最小请求，确认 **Flatkey** 已启用，并检查上述端点、密钥、余额和新会话步骤。 |

## 运行 Codex CLI

设置环境变量后，照常使用 Codex：

```bash
codex "Refactor this function to use async/await"
```

或进入交互模式：

```bash
codex
```

所有请求都会通过 Flatkey 路由，并按 Flatkey 优惠价格从预付余额中扣费。

## 选择模型

Codex CLI 默认使用 GPT 模型。你可以通过 `--model` 参数指定其他模型：

```bash
codex --model gpt-4o "Write unit tests for this module"
codex --model gpt-4o-mini "Add docstrings to this file"  # cheaper for simple tasks
```

请从[模型目录](/zh/reference/model-list)中选择当前列出且兼容 Codex 的文本模型，并复制其准确 ID。发送请求后，请在[用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)中确认实际使用的模型。图像和音频模型不适用于 Codex CLI。

## 验证手动环境变量配置

本节仅适用于手动设置 `OPENAI_BASE_URL` 和 `OPENAI_API_KEY` 的方法。如果你使用 CC Switch 配置 Codex，请改为按照上方的[重启并验证](#6-重启并验证)步骤操作。使用手动方法时，请运行 Codex 命令并查看[用量日志](https://console.flatkey.ai/usage-logs/common?lng=zh)。你应当能看到包含模型和 token 数量的请求记录。如果没有记录，请确认当前 shell 会话中的两个变量均已正确设置。

<Tip>
  如果持续使用 Codex，可选择 $200 充值档位获得更优惠的实际费率。叠加奖励后，费用最低可至 GPT 官方价格的 5 折。
</Tip>

## 手动环境变量配置故障排查

下表仅适用于手动设置 `OPENAI_BASE_URL` 和 `OPENAI_API_KEY` 的方法。如果你使用 CC Switch 配置 Codex，请查看上方的 [CC Switch 故障排查](#cc-switch-故障排查)。

| 问题 | 解决方法 |
|---|---|
| `Authentication error` | 确认 `OPENAI_API_KEY` 已设为以 `sk-fk-` 开头的 Flatkey 密钥 |
| `Model not found` | 在[模型目录](/zh/reference/model-list)中核对模型 ID |
| `Insufficient balance` | 前往 [console.flatkey.ai](https://console.flatkey.ai?lng=zh)充值余额 |
| 控制台中没有请求记录 | 确认当前 shell 中已设置 `OPENAI_BASE_URL` |
